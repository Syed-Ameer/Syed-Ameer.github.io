5 · Abbreviations & terms used in this document

Core
A major exam domain - a significant share of exam questions comes from this topic. Expect deep, scenario-based questions.
Covered
Included in the exam outline, but as part of a larger domain rather than a domain of its own.
Partial
Only a portion of the topic is tested. Example: AI-200 tests databases, but Cosmos DB / PostgreSQL / Redis - not Azure SQL.
Gap
Not tested in any of the three exams - learn it through Microsoft Learn / Applied Skills instead.
GA
General Availability - fully released to everyone, out of beta/preview. Normal booking, pricing, and scoring.
Beta
Pre-release phase of a new exam - limited seats, discounted, results delayed until rescoring completes.
CI / CD
Continuous Integration / Continuous Delivery - automated build, test, and release pipelines.
IaC
Infrastructure as Code - defining Azure resources in code (Bicep, ARM templates) instead of manual portal clicks.
ARM
Azure Resource Manager - Azure's deployment engine; ARM templates are its JSON-based IaC format (Bicep compiles to ARM).
RBAC
Role-Based Access Control - assigning permissions through roles rather than individual users.
KQL
Kusto Query Language - the query language for Azure Monitor logs and Application Insights telemetry.
SDK
Software Development Kit - the client libraries used to call Azure services from code.
OPDG
On-Premises Data Gateway - the bridge that lets Fabric/cloud pipelines reach on-premises data sources (successor role to ADF's self-hosted Integration Runtime).
T-SQL
Transact-SQL - Microsoft's SQL dialect, used in SQL Server, Azure SQL, and Fabric warehouses.
DW
Data Warehouse - the analytics warehouse workload in Microsoft Fabric.
ADF
Azure Data Factory - Azure's data integration service; its concepts now continue in Fabric Data Factory.
App Insights
Application Insights - Azure's application performance monitoring and telemetry service.
